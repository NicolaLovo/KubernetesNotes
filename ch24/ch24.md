# Chapter 24 -  Clusterrole and Clusterrole Binding

Roles are namespaced resources (namespace scoped),

Namespace resources (namespace scoped): pods, deployments... -> mean that they can be accessed only from inside their namespace

- `kubectl api-resources namespaced=true` -> see all types of namespaced resources
- roles are applied to namespace resources

> [!NOTE]
> If I do not specify a namespace to a role, it can be accessed from any namespace

Cluster scoped resources:

- namespaces
- nodes
- cluster role
- `kubectl api-resources namespaced=false` -> see all types of cluster resources

Cluster role:

- manages nodes, namespaces -> cluster-wide resources
- similarly, needs a clusterRole binding to be applied to a user/group of users

## Cluster roles

`kubectl auth can-i get noes --as user` -> check if I can access a cluster resource

`kubectl create clusterrole NAME --verbo=VERB --resource=RESOURCE` -> imperative cluster role creation

- `kubectl create clusterrole node-reader --verbo=get,list,watch --resource=node`

`kubectl get clusterrole`

`kubectl describe clusterrole NAME`

Create a cluster role binding: `kubectl create clusterrolebinding NAME --clusterrole=NAME --user=USERNAME --group=GROUPNAME`

- `kubectl create clusterrolebinding reader-binding --clusterrole=node-reader --user=adam`
