# Chapter 18 - Configmap 

## Configmap

Use case:

- we have lots of environment variables, maybe the same environment value is used in many deployments

Configmap: extract key-value pairs in an external object and inject it in the deployment

Imperative configmap creation:

- `kubectl create configmap app-cm --from-literal=NAME=VALUE` -> create configmap in an imperative way
  - es. `kubectl create configmap app-cm --from-literal=val1=hello --from-literal=val2=hello2`
- `kubectl create configmap app-cm --from-file=MY_FILE` -> create a config file from a config file
  - `kubectl create configmap app-cm --from-file=app.conf`

Declarative configmap creation:

- `kubectl create configmap app-cm --from-literal=NAME=VALUE --dry-run=client -o yaml > cm.yaml`

2 ways to mount a configMap:

1. load as a environment variable -> useful if there are few environment variables
2. load all the configMap in environment variables -> useful if there are lots of env variables
3. mount as a volume
